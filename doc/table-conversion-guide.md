# 표 변환 가이드 (MD 변환용)

## 1단계: 표 유형 판단

```
표 확인
├── 병합 셀 없음 → [A] 표준 MD 표 변환
├── colspan만 있음 → [B] =CS= 마커 변환
├── rowspan만 있음 → [B] =RS= 마커 변환
├── colspan + rowspan 단순 조합 → [B] 혼합 마커 변환
└── colspan + rowspan 동시 병합 (한 셀이 양방향) → [C] HTML 변환
```

---

## [A] 표준 MD 표 변환

병합 셀이 없는 일반 표는 표준 MD 문법으로 변환한다.

```markdown
| 헤더1 | 헤더2 | 헤더3 |
|-------|-------|-------|
| 내용  | 내용  | 내용  |
```

검증 루틴 생략 가능 (병합 없으므로)

---

## [B] 병합 마커 변환 규칙

### 마커 정의

| 마커 | 의미 | 조건 |
|------|------|------|
| `=RS=` | 위 셀의 rowspan 연속 (사라진 셀) | 같은 열 바로 위에 내용 셀 또는 `=RS=` 존재 |
| `=CS=` | 왼쪽 셀의 colspan 연속 (사라진 셀) | 같은 행 바로 왼쪽에 내용 셀 또는 `=CS=` 존재 |

- 대소문자 구분 없음 (파서 기준)
- ML 학습 데이터 생성 시 대문자 `=RS=` / `=CS=` 로 통일

### 변환 예시

**원본 (rowspan + colspan 혼합):**

```
| 구분     | A항목        | B항목 | C항목 |
|----------|---(colspan)--|-------|-------|
| 카테고리 | 내용1        | =CS=  | 내용2 |  ← A항목이 2열 병합
| =RS=     | 내용3        | 내용4 | 내용5 |  ← 구분이 2행 병합
| 소계     | 합계         | 값1   | 값2   |
```

**변환 결과:**

```markdown
| 구분     | A항목 | =CS=  | B항목 | C항목 |
|----------|-------|-------|-------|-------|
| 카테고리 | 내용1 | =CS=  | 내용2 | 내용3 |
| =RS=     | 내용4 | 내용5 | 내용6 | 내용7 |
| 소계     | 합계  | 값1   | 값2   | 값3   |
```

### 3칸 이상 병합

연속 마커 수 + 1 = 병합 칸 수

```
colspan=3:  | 내용 | =CS= | =CS= |   (마커 2개 → 3칸)
rowspan=3:  행1: | 내용 |
            행2: | =RS= |
            행3: | =RS= |           (마커 2개 → 3행)
```

### 변환 절차 (행마다 반복)

```
① 원본 표의 첫 번째 행(헤더)부터 시작
② 왼쪽 → 오른쪽 순서로 각 셀 처리:
   - 병합으로 사라진 셀 → 방향에 따라 =RS= 또는 =CS= 입력
   - 내용 있는 셀 → 내용 그대로 입력
③ 행 작성 완료 후 → 검증 루틴 실행
④ 검증 통과 시 다음 행으로
⑤ 모든 행 완료 후 최종 확인
```

---

## 검증 루틴 (행마다 의무 실행)

### 1단계: 열 수 확인 (겠수 확인)

```
(내용 셀 수) + (=RS= 수) + (=CS= 수) = 전체 열 수
```

- 일치 → 2단계 진행
- 불일치 → 해당 행 재작성 후 재검증

### 2단계: 라벨 위치 확인 (라벨 확인)

```
=RS= 확인:
  → 같은 열의 바로 윗 행에 내용 셀 또는 =RS= 가 있는가?
  → YES: 통과 / NO: 오류, 해당 셀 수정

=CS= 확인:
  → 같은 행의 바로 왼쪽에 내용 셀 또는 =CS= 가 있는가?
  → YES: 통과 / NO: 오류, 해당 셀 수정
```

두 확인 모두 통과 → 다음 행으로 진행
오류 발견 → 해당 행 수정 후 검증 루틴 재실행

## [C] HTML 변환 (예외 케이스)

다음 경우 HTML 표로 작성한다:

1. 한 셀에 colspan + rowspan 동시 적용된 경우
2. 표 내용에 `=RS=` 또는 `=CS=` 텍스트가 실제로 포함된 경우
3. 병합 구조가 불규칙하여 마커 방식으로 표현 불가능한 경우

```html
<table>
  <tr>
    <td rowspan="2" colspan="2">내용</td>
    <td>내용</td>
  </tr>
  <tr>
    <td>내용</td>
  </tr>
</table>
```

## 전체 플로우

```
원본 표 입력
    ↓
유형 판단
    ├── 병합 없음 → [A] 표준 MD 변환 → 완료
    ├── 단방향 병합 → [B] 마커 변환
    │       ↓
    │   행 작성
    │       ↓
    │   [검증] 열 수 확인 → 불일치 시 재작성
    │       ↓
    │   [검증] 라벨 위치 확인 → 오류 시 수정
    │       ↓
    │   다음 행 반복
    │       ↓
    │   최종: 모든 행 열 수 동일 확인
    │       ↓
    │   완료
    └── 양방향 동시 병합 → [C] HTML 변환 → 완료
```

## 활용 목적별 출력 방식

| 목적 | 출력 방식 |
|------|----------|
| AI 중간 처리 (토큰 절약) | `=RS=` / `=CS=` 마커 MD |
| 최종 문서 출력 | 마커 → HTML 역변환 |
| Word/PPT 붙여넣기 | HTML 표 복사 |
| ML 학습 데이터 | 대문자 마커 통일 |
