# 서브젝트 가이드

## 범위
현재 대상 과목 전용 변환 절차, 챕터 분류, 예외 규칙을 정의한다.

## 기본 가정
- P-001 별도 지시/가이드가 없으면 입력 파일 1개를 렉처 1개로 간주한다.
- P-002 챕터명 표기는 변형될 수 있으므로 텍스트 형태보다 기능 의미로 분류한다.

## 변환 흐름(고정)
1. P-010 사전 고정: 공통/단어/서브젝트 가이드 버전을 확정
2. P-011 입력 확인: `raw.txt` 기준 변환 시작
3. P-012 챕터 경계 탐지: 렉처 내부 기능 단위(도입/본학습/활동/평가/정리 계열) 식별
4. P-013 챕터 분류: 필수/선택 챕터 분류 및 조건 확인
5. P-014 우선순위 확정: 선택 규칙 충족 여부를 먼저 판단
6. P-015 구조/라벨 변환 수행
7. P-016 단어 정규화 적용
8. P-017 최종 검수 및 산출
9. P-018 변경 관리: 새 규칙 발견 시 가이드에 추가하고 같은 렉처에 재적용

## 우선순위 규칙
- P-020 선택 규칙은 조건 충족 시 필수 규칙보다 우선한다.
- P-021 충돌 시에는 더 구체적인 조건 규칙이 상위 원칙을 덮어쓴다.
- P-022 실행 전 공통/단어/서브젝트 규칙 충돌을 비교한다.

## 챕터/패턴 운용
- P-030 교안 흐름 패턴은 `위계구조형/흐름형/문제해결형/사례중심형` 중 분류한다.
- P-031 본학습 외 챕터 구성은 과목 특성에 따른 변형을 허용한다.
- P-032 챕터명 변형은 사용자 피드백으로 확정 후 과정 규칙으로 승격한다.
- P-033 섹션 경계 잠금: `[학습활동]`/`[학습활동에 대한 의견]` 구간에서는 번호 패턴(`1.`, `1)`, `(1)`)을 자동으로 상위 제목으로 승격하지 않는다.
- P-034 섹션 복귀 조건: 활동/의견 구간에서 본학습 제목 패턴(예: `n. 대주제`)이 나오면 의견 블록에 병합하지 않고 본학습 위계로 복귀해 재판정한다.
- P-035 예시/오류목록 보호: `※ ... 예`, `... 오류 사례` 하위의 번호 목록은 제목이 아니라 목록 항목으로 유지한다.

## 라이브러리 업데이트/적용 플로우(고정)
1. P-050 시작 시점에 라이브러리 버전 `vN`을 고정한다.
2. P-051 구조/라벨 1차 변환 후 `vN`으로 단어 치환을 수행한다.
3. P-052 신규 단어는 즉시 본문 확정하지 않고 `후보목록`에 적재한다.
4. P-053 애매 단어는 사용자에게 질문해 표준형을 확정한다.
5. P-054 확정된 단어를 라이브러리에 반영해 `vN+1`을 만든다.
6. P-055 해당 렉처 전체에 `vN+1`을 재적용한다. (부분 적용 금지)
7. P-056 최종 산출물에는 마지막 버전 1개만 남긴다.
8. P-057 치환 발생 시에만 하단 치환 참고 테이블을 기록한다.

## 예외 처리
- P-040 애매한 경우 `원문/후보1/후보2/권장안` 형식으로 질문한다.
- P-041 확정된 예외는 재사용 규칙으로 저장해 동일 유형 재질문을 방지한다.
- P-042 논라벨 규칙은 본 가이드 범위에서 제외한다.
- P-043 구조 이상(의견 블록 내 본학습 제목 혼입, 번호 목록의 `##` 오승격)이 발견되면 즉시 2차 정제에서 재구조화 후 확정한다.
