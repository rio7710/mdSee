# 공통 가이드

## 범위
과목 공통의 변환 포맷 규칙을 정의한다. (라벨 유지본 기준)

## 최상위 원칙
- C-001 의미론적 구조화: 라벨과 제목 기준으로 내용을 분절/구조화한다.
- C-002 원문의 의미/사실/맥락은 보존한다.
- C-002-1 원문에 없는 인과/판단/추론 표현(예: '관찰 결과를 바탕으로')을 임의 추가하지 않는다.
- C-002-2 원문의 제한/예외/주의(예: '용이하지 않음', '효과적이지 않을 수 있음') 문장은 축약·삭제하지 않고 보존한다.
- C-003 포맷 규칙과 원문 구조 신호가 충돌하면 원문 구조를 우선 보존한다.
- C-004 기본 편집 원칙은 원문 유지이며, 수정은 문장 삭제/재서술보다 분절·위계 정리에 우선 배분한다.

## 핵심 단위
- 과정: 여러 렉처(차시/강)의 집합
- 렉처: 30~60분 단위의 단일 교육 회차
- 챕터: 렉처 내부 진행 단위(도입/본학습/활동/평가/정리 계열 포함)
- 항목: 챕터 내부 최소 설명 단위(필요 시 재귀 확장)
- 콘텐츠: 항목의 말단에 붙는 라벨 단위(`[정의]`, `[해설]` 등)

## 공통 위계 기준
- 과정 > 렉처 > 챕터 > 항목(재귀 가능) > 콘텐츠(라벨)
- 본학습은 위계 레벨이 아니라 챕터의 한 유형으로 취급한다.

## 위계 구조화 규칙
- C-010 렉처 단위의 최상위 챕터 제목은 `##` 레벨로 고정한다.
- C-011 본학습 대주제는 로마자 번호를 유지해 `## I. ...`, `## II. ...` 형식으로 변환한다.
- C-012 대주제의 1차 하위 주제는 `###`로 변환하고, 숫자 표기는 `1.`, `2.`를 기본으로 통일한다.
- C-013 2차 하위 설명/세부 항목은 `####`로 변환하며, `1)`, `2)` 표기를 유지해도 위계가 유지되면 허용한다.
- C-013-1 `####` 제목이 서술문 종결(`~다.`, `~이다.`, `~것이다.`, `~아니다.`)을 포함하면 제목/본문 혼합으로 간주한다. 제목은 개념명·항목명으로 축약하고, 서술 내용은 바로 아래 라벨 본문(`[정의]` 또는 `[해설]`)으로 분리한다.
- C-014 독립 블록(표/활동/계획서/평가 블록 등)은 문장 블릿으로 두지 않고 최소 ### 이상 제목으로 승격한다. 목차 블록은 본문 대주제와 1:1 매핑을 유지한다.
- C-014-1 '별첨 자료' 계열 챕터의 번호/명칭은 원문 표기를 우선 고정한다. (요약형 제목으로 일반화하지 않음)

## 라벨 규칙
- C-020 기본 라벨 세트: `[정의] [해설] [사례] [근거] [인용] [질문] [실습] [요약] [전환]`
- C-021 콘텐츠 라벨은 가능한 말단 설명 단위에 배치한다.
- C-022 한 문단 1라벨 원칙을 우선한다. 복합 내용은 문단 분리 후 라벨링한다.
- C-022-1 분절된 하위 항목(번호/블릿 포함)은 상위 라벨을 암묵 상속하지 않고, 각 항목에 라벨을 명시한다.
- C-023 `정의/해설` 판정은 조사(`은/는/이/가`) 단독 기준으로 결정하지 않는다.
- C-024 `[정의]` 우선 패턴: `A는 B이다/임/것이다`, `A란/이란`, `용어: 개념 규정`처럼 용어의 본질을 규정하는 문장.
- C-025 `[해설]` 우선 패턴: 원인/목적/효과/맥락/절차를 설명하는 문장.
- C-026 콜론(`:`) 문장 판정: `용어: 개념 정의`면 `[정의]`, `항목명: 운영/주의/절차 설명`이면 `[해설]`.
- C-027 경계 사례는 `[정의]`로 단정하지 말고 `[해설]`로 두고 사용자 확인 후 확정한다.
- C-028 번호형 항목 판정은 표면 형식(`(1)`, `①`)만으로 결정하지 않고, 항목의 기능(용어 규정 vs 운영 설명)으로 판정한다.
- C-028-1 `[정의]` 조건: 번호형 항목이 특정 개념/기법/유형의 이름을 제시하고 그 의미·성격을 규정하면 `[정의]`로 부여한다. (콜론 유무 무관)
- C-028-2 `[해설]` 조건: 번호형 항목이 절차, 주의점, 비교, 적용 방법, 효과/한계를 설명하면 `[해설]`로 부여한다.
- C-028-3 혼합 항목 처리: 제목부는 `[정의]`, 후속 설명 블릿은 내용 기능에 따라 `[해설]` 또는 `[사례]/[근거]`로 분리한다.
- C-028-4 예외 처리: 판정이 애매하면 `[해설]` 임시 부여 후 사용자 확인으로 확정한다.

## 해설/문장 스타일
- C-030 `[해설]` 어미는 `~함/~됨/~임` 또는 명사형으로 통일한다.
- C-030-1 보조 문장이 상태/필요성/원칙을 설명하는 경우 `~함`을 강제하지 않고 명사형 종결을 허용한다.
- C-030-2 본문 해설 문장도 의미 변화가 없으면 서술형(`~한다`)을 명사형(`~함` 또는 핵심 동사/명사형)으로 축약할 수 있다. 예: `...제공한다` → `...제공`.
- C-030-3 `학습하기(본학습)` 구간의 `[해설]`은 인용문을 제외하고 명사형/줄임동사 어미를 기본으로 강제한다. (`~함/~됨/~임/핵심 동사`)
- C-031 `[학습목표]`는 관찰 가능한 행동 동사 문장으로 작성하고 마침표를 유지한다.
- C-032 일반 서술에서 불필요한 종결 마침표는 제거한다. (`[학습목표]` 예외)
- C-033 긴 결합 문장은 의미 단위로 분절하며, 상위 문장을 보조하는 하위 문장은 `[해설] § ...` 형식으로 표시한다.
- C-033-1 분절 결과가 원문의 인과/조건/대상/범위를 바꾸면 해당 분절은 금지한다.
- C-034 `제목: 설명` 구조는 화살표 분절 대신 일반 하위 블릿으로 처리한다.
- C-035 원문이 줄바꿈으로 이미 분리된 독립 문장은 `§` 보조 분절을 강제하지 않는다.
- C-036 `주의할 점`, `선택 기준`, `구성 요소`처럼 항목 나열을 예고하는 명사형 표제 문장은 동사형 종결(`~함`)로 재작성하지 않고, 표제 + 하위 나열 구조를 유지한다.
- C-037 하위 항목이 없는 긴 설명문은 `핵심어(표제)`와 `설명`으로 1회 분절할 수 있다. 단, 이미 하위 항목이 있는 상위 문장은 추가 분절하지 않는다.
- C-038 쉼표/접속어로 연결된 동등 절은 의미가 같은 위계이면 같은 깊이의 블릿으로 분절할 수 있다. 보조 문장이 1~2개면 `[해설] § ...`를 사용하고, 보조 문장이 많아지면 일반 하위 블릿으로 전환한다.
- C-039 `**항목명**: 요약문` 형태의 상위 설명 뒤에 이어지는 보강 문장은 상위 항목의 하위 뎁스(`  -`)로 배치한다. 같은 의미군의 보강 문장은 동일 하위 레벨을 유지한다.
- C-039-1 문장 내부 마침표(`.`) 기준 분절은 의미가 달라지지 않는 경우에만 허용한다. 의미가 달라지는 분절은 금지한다. 독립 의미면 동일 뎁스, 보충 의미면 하위 뎁스로 배치한다.

## 특수 콘텐츠 포맷
- C-040 `[근거]`는 인용구 블록(`> **[근거]**`)으로 분리한다.
- C-041 `[사례]`는 인용구 블록(`> **[사례]**`)으로 분리한다.
- C-041-1 외부 문장/원전 문구를 직접 전거와 함께 제시하는 경우 `[인용]` 라벨을 사용한다.
- C-041-2 인용구 블록(`> ...`)과 `[인용]` 라벨 문장은 원문 유지 원칙을 적용한다. (띄어쓰기/문장부호/어미 임의 보정 금지)
- C-042 `[그림]`은 인용구 블록과 함께 텍스트 설명을 보강한다.
- C-043 `[평가하기]`는 정답/해설을 분리해 `**정답:**`, `**해설:**` 형식으로 통일한다.
- C-044 제목/타이틀이 `표`로 식별되는 블록(예: `<표1>`, `표 2`)은 목록이 아닌 마크다운 표로 구성한다.

## 렌더링 분리 원칙
- C-050 마크다운 원문은 구조 데이터로 취급한다.
- C-051 박스/색상/스타일은 렌더링 후처리 단계에서 적용한다.
- C-052 표 셀 내부는 단순 텍스트와 `<br>` 중심으로 유지한다.
- C-053 HTML 블록/인라인 CSS/렌더러 종속 문법은 기본 금지한다.

## 기본 작업 플로우
- C-070 1차 정제: 가이드 규칙에 따라 구조화/라벨링/분절/표준단어 치환을 수행한다.
- C-070-1 1차 정제 완료 후 즉시 사용자 확인 게이트를 수행한다. 요약(주요 변경점/의심 구간/예상 리스크)을 제시하고 `이대로 2차 진행할까요?`를 질문한다.
- C-070-2 사용자 승인(주문/지시) 전에는 2차 정제·최종 확정을 진행하지 않는다.
- C-071 원문 대조: (사용자 승인 후) 1차 정제본을 원문(TXT)과 비교하여 의미/사실/맥락 변화 여부를 점검한다.
- C-072 2차 정제: 대조 결과를 반영해 의미 보존 우선으로 보정한다. (과도한 재서술보다 원문 복원/분절 조정 우선)
- C-073 최종 검수: 라벨/위계/표기 통일/예외 규칙(인용문 원문 유지 포함) 적용 여부를 확인하고 최종본 1개를 확정한다.
- C-074 2차 정제 완료 후 간단 리스크 레포트를 작성한다. 최소 항목: `리스크`, `예상 문제`, `발생 조건`, `완화/대응`, `추가 확인 필요`.
- C-075 리스크 레포트는 `mlDoc/정제_리스크_레포트.md`에 누적 기록한다.
- C-076 레포트 헤더 식별자는 `YYYY-MM-DD | 교안명` 형식으로 기록한다.
- C-077 레포트 저장 후 현재 작업 창에도 동일 내용을 3~5줄로 요약 제시한다.

## 공통 검수 체크리스트
- C-060 라벨 누락 여부
- C-061 위계 붕괴 여부
- C-062 번호 체계 혼용 여부
- C-063 중복 목록 마커 잔존 여부
- C-064 `§` 보조 표기 오용 여부
- C-065 잔여 마크업/리터럴 개행 문자열 잔존 여부
- C-066 표 타이틀 블록의 표 구성 여부 확인
- C-067 `학습활동/의견` 블록 내부 본문 제목 혼입 여부 확인
- C-068 번호 목록(`1.`, `1)`, `(1)`)의 제목 오승격(`##/###`) 여부 확인
- C-069 `학습활동` 블록은 `질문 -> 의견 -> 해설 항목` 계층을 유지한다. `의견:`은 독립 문장이 아닌 부모 항목(`- **의견:**`)으로 두고, 후속 `[해설]`은 반드시 하위 뎁스(`  -`)로 배치한다.
- C-069-1 `정리하기`의 `[요약]`은 번호 체계의 의미 위계를 실제 마크다운 뎁스로 반영한다. 상위 번호(`1.`, `2.`)를 부모로 두고 하위 번호(`1)`, `(1)`)는 각각 한 단계씩 하위 뎁스로 배치한다. 표면 번호만 있고 같은 뎁스로 평면 배치하는 것을 금지한다.
