# 단어 가이드

## 범위
사회복지 과목의 띄어쓰기/결합형/기호/표현 편차를 통일한다.

## 운영 원칙
- W-001 사용자가 추가한 단어는 우선 표준으로 채택한다.
- W-002 표준형이 애매하면 적용 전에 사용자에게 질문한다.
- W-003 의미 왜곡, 반대 의미 전환, 과교정 위험이 있으면 자동 치환하지 않는다.
- W-004 치환은 라벨 종류와 무관하게 문서 전체에 적용한다.

## 적용 시점
- W-010 구조/라벨 변환 완료 후, 최종 검수 전에 적용한다.
- W-011 새 패턴 발견 시 규칙을 추가하고 현재 렉처에 즉시 재적용한다.

## 규칙 구역
- 자동 적용: 안전하고 승인 완료된 치환
- 확인 후 적용: 애매하여 사용자 확인이 필요한 치환
- 치환 금지: 문맥 훼손 위험이 있어 자동 치환 금지

## 표기 통일 규칙
- W-020 결합형 단어 통일 (`개발 단계` -> `개발단계` 등)
- W-021 병렬 결합 표현의 점(`.`) 표기는 의미 문장부호(`·`)로 교정
- W-022 의미 분절 표시는 `[해설] § ...` 형식으로 통일한다. (`?`, `→`, `▷`, `->`, `[보조]` 직접 사용 금지)

## 적용 결과 기록
- W-030 치환이 발생한 경우에만 `*_최종.md` 하단에 `표준단어 라이브러리 적용 참고` 테이블을 기록한다.
- W-031 실제 치환된 항목만 `원문 표현 / 표준 표현 / 적용 위치`로 기록한다.
- W-032 치환이 없는 경우 테이블을 생성하지 않는다.

## 라이브러리 포맷 권장
- W-040 포맷은 YAML을 기본으로 사용한다. (사람 검토 + 버전관리 diff 용이)
- W-041 상단 메타 필드: `version`, `updated_at`, `owner`, `scope`
- W-042 규칙 구역을 파일 구조에 그대로 반영한다: `auto_apply`, `ask_first`, `do_not_replace`
- W-043 항목 필드 권장: `id`, `source`, `target`, `context`, `note`

## YAML 예시
```yaml
version: 1.2.0
updated_at: 2026-02-24
owner: 이수정
scope: 사회복지

auto_apply:
  - id: W-001
    source: 개발 단계
    target: 개발단계
    context: 개발단계별 평가
    note: 결합형 통일

ask_first:
  - id: W-101
    source: 사례 관리
    target: 사례관리
    context: 사례 관리 회의
    note: 기관 문서마다 분리 표기 관행 존재

do_not_replace:
  - id: W-201
    source: 기능
    target: null
    context: 기능/기능적/역기능
    note: 문맥 의존성이 커서 자동 치환 금지
```

|---|---|---|---|---|
| W-001 | 개발 단계 | 개발단계 | 개발단계별 평가 | 자동 적용 |
